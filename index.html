发表于 2020-08-04 分类于 数据结构与算法

树

定义

树（tree）是包含n（n>=1）个结点，(n-1)条边的有穷集，其中：
（1）每个元素称为结点（node）；
（2）有一个特定的结点被称为根结点或树根（root）。
（3）除根结点之外的其余数据元素被分为m（m≥0）个互不相交的集合T1，T2，……Tm-1，其中每一个集合Ti（1<=i<=m）本身也是一棵树，被称作原树的子树（subtree）。
树也可以这样定义：树是由根结点和若干颗子树构成的。树是由一个集合以及在该集合上定义的一种关系构成的。集合中的元素称为树的结点，所定义的关系称为父子关系。父子关系在树的结点之间建立了一个层次结构。在这种层次结构中有一个结点具有特殊的地位，这个结点称为该树的根结点，或称为树根。
（以上来源于 百度百科）

种类

无序树：树中任意节点的子结点之间没有顺序关系，这种树称为无序树,也称为自由树;
有序树：树中任意节点的子结点之间有顺序关系，这种树称为有序树；
二叉树：每个节点最多含有两个子树的树称为二叉树；
满二叉树：叶节点除外的所有节点均含有两个子树的树被称为满二叉树
完全二叉树：有个 2^k -1 节点的满二叉树称为完全二叉树
哈夫曼树（最优二叉树）：带权路径最短的二叉树称为哈夫曼树或最优二叉树；
（以上来源于 百度百科）

首先声明一下二叉树结点的定义

二叉树作为一种特殊的树，也有三种不同的遍历方式：前序，中序，后序遍历。其中，中序遍历是二叉树所特有的遍历方式。
在离散数学中学过，对于三种遍历方式，给出前，中，后序的遍历结果，只有 前序 和 后序 的遍历结果可以唯一地对应一棵二叉树，而 中序 的遍历结果则可能对应多棵二叉树

递归算法

递归算法是最简单最简洁的遍历算法啦，三种遍历方式只是操作输出语句的位置不同而已哈哈哈哈，做力扣的二叉树遍历问题直接用递归写简直不要太爽嘻嘻！
但是呢，递归算法牺牲了内存中的系统栈，当递归深度过大时，栈内存可能就不够用啦。而且每次调用函数，都会首先在系统栈中压入返回地址（将要执行的下一条指令的地址），可作为栈溢出的攻击对象，很不安全。但是…它好写呀，好理解呀！！这不香嘛？？？

迭代算法

好了，真正的重头戏来了。虽然迭代算法需要各种标记，各种栈，对我这种新手非常的不友好，但是它依然有他的应用场景，也会有很多地方需要迭代地遍历二叉树，也需要我们掌握。
迭代算法的本质，其实也是模拟递归算法，递归调用了系统栈，而迭代算法呢，则用stack（STL）来模拟栈

前序遍历

首先，创建stack存放结点。首先想先打印根结点，那就先把根节点压入栈中，再想想，想先打印左子树，再打印右子树，那么就要先把右子树压入栈，再左子树入栈。

中序遍历

首先，还是创建一个栈stack,按左 中 右的顺序输出结点。还是上来就把根结点压入栈呀。
然后就要尽可能地把左子树压入栈，只是入栈还不行，同时还要及时处理，也就是输出呀，左子树也有一颗颗的小的子树都要处理。那就一直向下，不断入栈，知道最左边的结点。此时这个结点就在栈顶了。其目的是找到一个最小单位的子树(也就是最左侧的一个节点)，而且还记录了来源（就是在它之前压入栈的结点），这样才能返回上层对吧。而且！当我们返回上层的时候，那颗小的左子树就处理完了，接下来就那个上层结点的右子树了对不对。再处理完了就返回更上一层，balabala…依次这样处理了
当处理完最小单位的子树时，返回到上层处理了中间节点。如果把整个左中右的遍历都理解成子树的话，就是处理完 左子树->中间(就是一个节点)->右子树）
如果有右节点，其也要进行中序遍历。

在leetcode里，后序遍历的标签是“困难”，也真是最难理解的了（对于小菜鸡lzj来说呜呜呜）。

题目：

给出两个 非空 的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储 一位 数字。

如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。

您可以假设除了数字 0 之外，这两个数都不会以 0 开头。

Solution

这道算法题考察的是数据结构中的链表的操作。我写的算法比较复杂啦，但是思路应该是一致的，都是人们很容易想到的，正常的算术方法:逐位计算，进位…

这是链结ListNode的定义，是一结构体。

弱鸡lzj的代码

l1,l2是对应操作数链表的头指针,l3是运算结果的头指针。a,b对应两个操作数指向位的指针，c对应运算结果指向位的指针。a,b,c指针指向当前运算的某一位，在本位计算完成后，
a,b移向下一位，c->next则重新new一个链结，并初始化为0，将c指针指向该链结。

而这里的难点在哪里呢？ 也是困扰我很久的地方，debug十几次都不对。。正是操作数的加法情况较多，需要全面考屡。
1，两个操作数的位数不一定一样对吧 2，产生的进位怎么处理呢？有可能一直向下产生进位的呦…

1.比如[0,1,2,5]+[1,2] [5]+[5]
2.比如[9,9,9]+[1] 这个就非常讨厌，不好想到

代码中第一个循环就是处理位数重合部分的，一直算到比较短的那个操作数不能再往后读啦！那怎么办咧？进位已经产生啦，就只好加到那个位数较多的数字的下一位咯…
这里我很笨拙地讨论了三种情况，分别是l1比l2长，l2比l1长，以及二者一样长…因为有[9,9,9]+[1]这种情况存在，可能每一位都产生进位，所以在算完短操作数的全部位之后，不能把长操作数多出的那些位直接平移到答案（ans）链表中哦！！

l1,l2长度不同时，处理的算法是一样的，不同的是l1,l2长度一样该怎么办咧？很简单，在计算完最后一位后，可能会产生进位(0或1),那就把这个进位加到c->next所new出来的一个结点就好啦！！

但是新的问题又出现了，笨拙的三种讨论，都会有一个共同的问题，就是算到最后一位时，最后一位(也即最高位)如果产生的进位是1还好，l3的c指针的c->next又new出一个结点，正好装了那个进位1。如果是0呢? 意思是答案最高位是0？ 就该清掉最高位的0啦！

所以最后一步是遍历计算结果的链表，判断最高位，也就是计算结果链表的尾结点的Val是否为0。为0，则把尾结点delete掉，它的上一个结点的next设为NULL就好啦啦啦啦
