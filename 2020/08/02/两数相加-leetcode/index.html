两数相加 ---- leetcode

发表于 2020-08-02 更新于 2020-08-04 分类于 leetcode

题目：

给出两个 非空 的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储 一位 数字。

如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。

您可以假设除了数字 0 之外，这两个数都不会以 0 开头。

Solution

这道算法题考察的是数据结构中的链表的操作。我写的算法比较复杂啦，但是思路应该是一致的，都是人们很容易想到的，正常的算术方法:逐位计算，进位…

这是链结ListNode的定义，是一结构体。

弱鸡lzj的代码

l1,l2是对应操作数链表的头指针,l3是运算结果的头指针。a,b对应两个操作数指向位的指针，c对应运算结果指向位的指针。a,b,c指针指向当前运算的某一位，在本位计算完成后，
a,b移向下一位，c->next则重新new一个链结，并初始化为0，将c指针指向该链结。

而这里的难点在哪里呢？ 也是困扰我很久的地方，debug十几次都不对。。正是操作数的加法情况较多，需要全面考屡。
1，两个操作数的位数不一定一样对吧 2，产生的进位怎么处理呢？有可能一直向下产生进位的呦…

1.比如[0,1,2,5]+[1,2] [5]+[5]
2.比如[9,9,9]+[1] 这个就非常讨厌，不好想到

代码中第一个循环就是处理位数重合部分的，一直算到比较短的那个操作数不能再往后读啦！那怎么办咧？进位已经产生啦，就只好加到那个位数较多的数字的下一位咯…
这里我很笨拙地讨论了三种情况，分别是l1比l2长，l2比l1长，以及二者一样长…因为有[9,9,9]+[1]这种情况存在，可能每一位都产生进位，所以在算完短操作数的全部位之后，不能把长操作数多出的那些位直接平移到答案（ans）链表中哦！！

l1,l2长度不同时，处理的算法是一样的，不同的是l1,l2长度一样该怎么办咧？很简单，在计算完最后一位后，可能会产生进位(0或1),那就把这个进位加到c->next所new出来的一个结点就好啦！！

但是新的问题又出现了，笨拙的三种讨论，都会有一个共同的问题，就是算到最后一位时，最后一位(也即最高位)如果产生的进位是1还好，l3的c指针的c->next又new出一个结点，正好装了那个进位1。如果是0呢? 意思是答案最高位是0？ 就该清掉最高位的0啦！

所以最后一步是遍历计算结果的链表，判断最高位，也就是计算结果链表的尾结点的Val是否为0。为0，则把尾结点delete掉，它的上一个结点的next设为NULL就好啦啦啦啦
(我知道这一步看起来非常臃肿，小菜鸡还是要继续努力呀，写出更好的代码，向大牛学习！！！)
